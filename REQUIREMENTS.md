# 音声編集アプリ 要件定義書

## 1. 概要

ローカルPC（Windows）にインストールして使う、デスクトップ音声編集アプリケーション。
波形を見ながら不要な部分をカットし、音量を調整して書き出すことを目的とする。

## 2. 対象ユーザー・利用環境

- **OS**: Windows のみ
- **配布形態**: インストーラー（.exe / NSIS 形式）でローカルインストール
- **オフライン動作**: インターネット接続なしで全機能が動作すること

## 3. 対応フォーマット

### 3.1 読み込み（入力）
- MP3
- WAV
- M4A（AAC）

### 3.2 書き出し（出力）
- 書き出し時にユーザーが形式を選択できる
- 選択肢: MP3 / WAV / M4A

> **技術メモ**: WAV・MP3 の読み込みはブラウザ標準（Web Audio API）で対応可能。
> M4A の読み込み、および MP3 / M4A での書き出しには ffmpeg の同梱（ffmpeg-static など）が必要。
> アプリ内に ffmpeg バイナリをバンドルし、オフラインで変換できるようにする。

## 4. 機能要件

### 4.1 波形表示
- 読み込んだ音声の波形を画面に表示する
- 再生・停止・一時停止、シーク（波形上のクリックで再生位置移動）
- 再生位置を示すカーソル（プレイヘッド）を表示

### 4.2 範囲選択とカット
- 波形上をドラッグして編集対象の範囲を選択できる
- **複数箇所**の選択・カットに対応する
- 選択範囲をカットすると、前後の音声が結合され、波形が再描画される
- カット範囲の始点・終点を微調整できると望ましい

### 4.3 音量調整
- **音声全体**への音量調整に対応する
- **選択した範囲のみ**への音量調整に対応する
- スライダー等で調整し、書き出し前にプレビュー再生で確認できる

### 4.4 アンドゥ・リドゥ
- カット・音量調整などの編集操作を取り消し（Undo）・やり直し（Redo）できる
- 複数段階の履歴を保持する

### 4.5 書き出し
- 編集結果をファイルに保存する
- 保存ダイアログで、保存先・ファイル名・出力形式（MP3/WAV/M4A）を指定できる
- 元ファイルを上書きせず、別ファイルとして保存することを基本とする

## 5. 非機能要件・制約

### 5.1 長尺音声への対応（最重要）
- **最長 3 時間程度**の音声を扱えること
- 3 時間のステレオ音声は非圧縮で数 GB 規模になるため、
  「全データをメモリに展開して描画・処理」する素朴な実装は避ける
- 想定する設計方針:
  - **波形描画**: 全サンプルではなく、ダウンサンプリング済みの軽量ピークデータを生成して描画する
  - **カット・変換処理**: メモリ上でのバッファ結合ではなく、ffmpeg を用いてディスク上のファイルに対して処理する方式を優先検討する
  - **メモリ使用量**が長尺ファイルでも破綻しないことを確認する

### 5.2 編集対象
- ファイルは **1 つずつ**編集できれば十分（バッチ処理・複数同時編集は不要）

### 5.3 動作
- 一般的なスペックの Windows PC で、長尺ファイルでも実用的な速度で動作すること

## 6. 想定技術スタック（Claude Code への提案。変更可）

- **アプリ基盤**: Electron（Web 技術でデスクトップアプリを構築、Windows インストーラー生成）
- **ビルド**: Vite
- **波形表示・範囲選択**: wavesurfer.js + Regions プラグイン
- **音声処理**: Web Audio API（プレビュー・軽量処理）＋ ffmpeg（ffmpeg-static、変換・カット・書き出し）
- **パッケージング**: electron-builder（NSIS インストーラー）

## 7. スコープ外（今回は作らない）

- 複数ファイルのバッチ処理・同時編集
- フェードイン/アウト、ノイズ除去、エフェクト等の高度な加工
- Mac / Linux 版
- クラウド連携・オンライン機能
